## Configuration hierarchy

`application.yaml` is the entry point; it imports 9 profile YAMLs:

1. **actuator-configuration.yml** вЂ” Actuator endpoints, health, loggers, env, configprops
2. **ai-configuration.yml** вЂ” ONNX transformer model and tokenizer URIs for embeddings
3. **db-configuration.yml** вЂ” SQLite datasource, Hibernate dialect, ddl-auto
4. **documentation-configuration.yml** вЂ” JDK source download directory, doclet work/output directories
5. **ingest-configuration.yml** вЂ” Ingest auto-discovery configuration
6. **logging-configuration.yml** вЂ” Logback rolling policy, log file path
7. **mcp-configuration.yml** вЂ” Spring AI MCP server (name, streamable protocol)
8. **search-configuration.yml** вЂ” Hibernate Search Lucene backend (directory type, root, sync)
9. **springdoc-configuration.yml** вЂ” OpenAPI/Swagger UI toggles

All values use environment variable placeholders for flexibility.

## JSON documentation output

The `data/` directory holds versioned JSON documentation generated by the `JsonDoclet` via `DocumentationService`. Each
JDK version gets its own subdirectory with one `index.json`, `chunks.jsonl`, and per-type JSON files. Generated by
`javadoc` CLI invocation from `runJavadocDoclet()`.

## Database layer

The `domain/` package defines JPA entities persisted in SQLite via Hibernate:

- **`JdkVersion`** вЂ” JDK version metadata (version string, Adoptium tag, distribution)
- **`JdkDocChunk`** вЂ” Chunked Javadoc content with a vector embedding (`float[]`) for semantic search
- **`JdkDocElement`** вЂ” Structured Javadoc element (class, method, field) with metadata

The `repository/` package provides Spring Data JPA repositories for these entities. Hibernate Search maps the vector
fields via `@VectorField` (384 dimensions, cosine similarity) enabling kNN semantic search on `JdkDocChunk`.

The `model/converter/` package holds the `FloatArrayConverter`, a JPA attribute converter that maps `float[]` to SQLite
BLOB storage.

## Build output

Maven produces a Spring Boot fat JAR in `target/` and a doclet JAR in `doclet/`:

```
doclet/
в””в”Ђв”Ђ JAIDoc-doclet.jar
```

The doclet jar is used by `javadoc -docletpath` to generate JSON documentation via the `JsonDoclet`.
