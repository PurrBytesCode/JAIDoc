# Database — Persistence and Vector Search for JSON Javadoc

## Summary

The JSON Javadoc generated by the doclet is persisted in SQLite and indexed with Hibernate Search/Lucene for vector
search filtered by JDK version. Ingestion is an explicit step that does not modify the existing doclet generation
pipeline.

## Schema

```
JdkVersion (jdk_version)            1 ──< JdkDocElement (jdk_doc_element)   [rawJson structural JSON per type/package/module]
   version UNIQUE                   1 ──< JdkDocChunk   (jdk_doc_chunk)     [@Indexed: text + embedding + metadata]
                                              │ ManyToOne (optional)
                                              └──> JdkDocElement (parent element)
```

### Tables

| Table             | Description                                              |
|-------------------|----------------------------------------------------------|
| `jdk_version`     | JDK version metadata (version, counts, ingestion status) |
| `jdk_doc_element` | Raw structural JSON of each type/package/module          |
| `jdk_doc_chunk`   | Searchable chunks with vector embedding (384d)           |

### Main columns of `jdk_doc_chunk`

| Column          | Type    | Description                                       |
|-----------------|---------|---------------------------------------------------|
| `version`       | TEXT    | JDK version (denormalized, mandatory kNN filter)  |
| `chunk_id`      | TEXT    | Unique chunk identifier                           |
| `text`          | TEXT    | Chunk text                                        |
| `embedding`     | BLOB    | Vector embedding (384 floats, serialized as BLOB) |
| `kind`          | TEXT    | Element type (MODULE, PACKAGE, TYPE, etc.)        |
| `qualifiedType` | TEXT    | Full type name                                    |
| `packageName`   | TEXT    | Package it belongs to                             |
| `moduleName`    | TEXT    | Module it belongs to                              |
| `member`        | TEXT    | Member name (if it's a type member)               |
| `signature`     | TEXT    | Member signature                                  |
| `deprecated`    | BOOLEAN | Whether deprecated                                |

## Hibernate Search Mapping

- `JdkDocChunk` has `@Indexed` — automatically indexed by Hibernate Search
- The `embedding` field uses `@VectorField(dimension = 384, vectorSimilarity = COSINE)`
- Textual fields use `@FullTextField` and `@KeywordField` for filtering
- Synchronization is synchronous: `hibernate.search.default.indexing.plan.synchronization = sync`

## Index Directory

The Lucene index is stored on disk in the directory configured by `SEARCH_INDEX_DIR` (default `./jaidoc-index`).

## Configuration

Configured in `configurations/search-configuration.yml`:

```yaml
spring:
  jpa:
    properties:
      hibernate.search.backend.lucene.directory-type: local-filesystem
      hibernate.search.backend.lucene.directory.root: ${SEARCH_INDEX_DIR:./jaidoc-index}
      hibernate.search.default.indexing.plan.synchronization: sync
```

## Converter — FloatArrayConverter

Converts `float[]` ↔ `byte[]` BLOB. SQLite has no native vector type, so the embedding is serialized as a byte array
using `ByteBuffer`.

## EmbeddingService

Wraps the Spring AI transformer `EmbeddingModel` with e5 prefixes:

- `"passage: "` for embedding text (ingestion)
- `"query: "` for embedding queries (search)

Without these prefixes, ranking degrades silently.

## Ingestion

Ingestion is an explicit step (not yet implemented):

1. **Doclet pipeline** (current): JDK source → `JsonDoclet` → JSON Javadoc

The process:

- Verifies that `data/<version>/index.json` exists
- Deletes any prior ingestion of that version (idempotent)
- Parses the manifest → `JdkVersion`
- Loads structural JSON → `JdkDocElement`
- Reads `chunks.jsonl`, embeds and persists → `JdkDocChunk`

## Search

Search is restricted to one version at a time:

- **Service layer** (current): `JdkSearchService` with Hibernate Search kNN filtered by `version`
- **MCP tool** (planned): `searchJavadoc("25.0.3", "query", 10)`
- **REST endpoint** (planned): `GET /api/search?version=25.0.3&q=query&topK=10`

The kNN is filtered by `version` to guarantee isolation between versions.

## `ddl-auto` Notes

- `create-only` (default): tables are created at startup but not updated afterward. Need to delete
  `jaidoc.sqlite` between development iterations.
- `update`: Hibernate Search/ORM updates tables automatically during development.

## Dependencies

- Spring Data JPA
- Hibernate Search 8.4 (mapper-orm + backend-lucene)
- SQLite JDBC
- Hibernate community dialects
