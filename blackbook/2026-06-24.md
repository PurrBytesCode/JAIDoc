# 2026-06-24

### 💡 CPU for Search, GPU for Vector Creation — Model Split Strategy

- Realized CPU inference is suitable for the retrieval/search phase but a bottleneck during chunk vector creation
- Need to download the same transformer model in two variants:
    - CPU-optimized (smaller, sufficient for search)
    - GPU-optimized with Nvidia (larger model, better throughput for batch vectorization)
- Dependencies are already in place; next step is to wire up the dual-model path and document the process
- This will help anyone setting up the project — the GPU path dramatically reduces ingestion time

### 📝 Ingestion Performance — 167 Minutes on CPU

- Ingesting JDK 25.0.3 with the CPU-only model took **167 minutes 34 seconds** for 93,196 chunks
- This is too slow to be practical and confirms the need for GPU acceleration on vector creation
